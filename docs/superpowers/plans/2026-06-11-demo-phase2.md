# Phase 2 Demo Experience Implementation Plan ("ship the running process")

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Page 01's lifecycle card gains a **whole-VM lane**: a hot solver runs inside a real microVM, is frozen-and-published mid-flight with `plugin.publishMachine()`, pulled as a `machinen-vmstate@1` artifact, and resumes on a "second host" with its heap intact ÔÇö live on KVM machines, as a clearly-labeled recorded trace replay everywhere else. **Architecture:** Everything rides the revised plugin-owned runtime Phase 2 (spec: `docs/superpowers/specs/2026-06-11-image-federation-phase2-design.md`): the demo host's own `machinenPlugin({ driver: machinenDriver(), publish: {...} })` instance is the producer (no sidecar, no region agent), and a second MF instance in the same server process is the consumer, pulling `?artifact=vmstate` from the plugin's lazily-started artifact endpoint. Server-side capability detection (`/dev/kvm`, `@machinen/runtime`, platform) gates a live track vs a replay track fed by a committed trace fixture captured from a real KVM run. Demo design spec: `docs/superpowers/specs/2026-06-11-demo-phase2-design.md`.

**Tech Stack:** TypeScript (ESM, NodeNext) in `apps/host`; the Node guest (`apps/remote`); vanilla HTML/CSS/JS demo pages; `scripts/demo-web.mjs --smoke`; the IDE browser for visual verification.

**Conventions:** Build the host with `pnpm --filter host build` before running. Every number shown in the UI comes from real plugin hooks or the labeled replay fixture ÔÇö never synthetic. Server handlers follow the landed `server.ts` patterns: `routes` map, `wireStore.run` scoping, `json()`, `HttpError`, and the serialized-step helper. Page 01/02 markup is **in flux from the Phase 1 demo evolution** (sibling agent, plan: `docs/superpowers/specs/2026-06-11-demo-evolution-plan.md`); UI steps below are content specs ÔÇö integrate into whatever the landed cards/topology look like at execution time. ## Design adjustments vs the demo design doc (forced by the plugin-owned revision)

1. **Freeze and publish are one step.** The design doc's stepper had `2 ┬Ě freeze while running` and `3 ┬Ě publish vmstate bundle` as separate steps with a `POST /api/vm-demo/freeze` calling `snapshotMachine()`. The revised runtime owns snapshot-then-publish inside one verb, so the demo has one button (`freeze + publish`) backed by `publishMachine()`, and the stepper is four steps, not five.
2. **The demo kills the source explicitly.** The runtime's quiesce semantics resume the source VM after the dump. The hero moment needs "the source is gone", so the demo disposes the source VM after publishing ÔÇö an honest demo choice surfaced in the copy ("we chose to kill it; production sources keep running").
3. **No registry dir to stage, no second process.** The design doc's `/api/vm-demo/pull-restore` assumed a registry-shaped directory; now the consumer entry points at the plugin's own endpoint URL (returned by `publishMachine()`). Routes are renamed `/api/vm/*` and the demo gains zero new processes.
4. **Determinate progress bars are deferred to R4.** Phase 2a exposes only fetch start/end. The Task 6 transfer rail animates indeterminately between `beforeArtifactFetch` and `onArtifactFetched`, then shows the real bytes/ms; per-blob percent display is a marked follow-up step, blocked on R4.

```ts
// apps/remote/src/solver.ts
//
// The Phase 2 hero workload: a long-running route search with visible
// in-memory progress. Everything that matters lives in the heap ÔÇö the memo
// cache, the RNG state, the iteration count ÔÇö and NONE of it is covered by
// dehydrate(). An app-state snapshot loses it; a whole-VM vmstate snapshot
// moves it. That asymmetry is the demo.

const GRID = 64; // fixed search space so runs are comparable

// Mutable LCG: the RNG state itself is heap state. A restored VM continues
// the exact same sequence ÔÇö visible proof the heap moved, not a replay.
let rngState = 0x2026_0611;
function rng(): number {
  rngState = (rngState * 1664525 + 1013904223) >>> 0;
  return rngState / 0x1_0000_0000;
}

let running = false;
let iteration = 0;
let best = Number.POSITIVE_INFINITY;
const memo = new Map<string, number>(); // partial-path costs: the warm cache
let timer: NodeJS.Timeout | undefined;

function pathCost(x: number, y: number): number {
  const key = `${x},${y}`;
  const hit = memo.get(key);
  if (hit !== undefined) return hit;
  const cost = (Math.sin(x * 12.9898 + y * 78.233) * 43758.5453) % 1;
  const abs = Math.abs(cost);
  memo.set(key, abs);
  return abs;
}

function step(): void {
  // One bounded chunk per tick so the event loop (and /mf/call) stays live.
  for (let i = 0; i < 250; i++) {
    iteration++;
    let x = Math.floor(rng() * GRID);
    let y = Math.floor(rng() * GRID);
    let total = 0;
    for (let hop = 0; hop < 8; hop++) {
      total += pathCost(x, y);
      x = (x + 1 + Math.floor(rng() * 3)) % GRID;
      y = (y + 1 + Math.floor(rng() * 3)) % GRID;
    }
    if (total < best) best = total;
  }
}

/** FNV-1a over the values that define the heap: cheap, deterministic. */
function heapFingerprint(): string {
  let h = 0x811c9dc5;
  const mix = (n: number) => {
    h ^= n >>> 0;
    h = Math.imul(h, 0x01000193) >>> 0;
  };
  mix(iteration);
  mix(memo.size);
  mix(rngState);
  mix(Math.floor(best * 1e9));
  return h.toString(16).padStart(8, '0');
}

export interface SolverProgress {
  running: boolean;
  iteration: number;
  cacheSize: number;
  cacheCapacity: number;
  best: number;
  fingerprint: string;
  pid: number;
}

export function start(): SolverProgress {
  if (!timer) {
    running = true;
    timer = setInterval(step, 50);
    timer.unref();
  }
  return progress();
}

export function stop(): SolverProgress {
  if (timer) clearInterval(timer);
  timer = undefined;
  running = false;
  return progress();
}

export function progress(): SolverProgress {
  return {
    running,
    iteration,
    cacheSize: memo.size,
    cacheCapacity: GRID * GRID,
    best: best === Number.POSITIVE_INFINITY ? -1 : Math.round(best * 1e4) / 1e4,
    fingerprint: heapFingerprint(),
    pid: process.pid,
  };
}
```

Restore caveat the implementer must know: `setInterval` survives a vmstate restore because the whole event loop is in the snapshot ÔÇö that is the point. But `timer.unref()` matters: an unstarted solver must never keep an idle guest alive.

```ts
// apps/host/src/vm-capability.ts
//
// Server-side truth for the whole-VM demo lane. Same honesty contract as
// scripts/machinen-e2e.mjs: detect, never fake. The UI renders this verbatim.
import { accessSync, constants, existsSync } from 'node:fs';
import { createRequire } from 'node:module';

export type VmCapabilityReason =
  | 'live'
  | 'missing-kvm'
  | 'kvm-not-accessible'
  | 'unsupported-platform'
  | 'missing-runtime';

export interface VmCapability {
  available: boolean;
  reason: VmCapabilityReason;
  detail: string;
  platform: string; // OCI-style, e.g. linux/amd64
}

const OCI_ARCH: Record<string, string> = { x64: 'amd64', arm64: 'arm64' };

function ociPlatform(): string {
  return `${process.platform}/${OCI_ARCH[process.arch] ?? process.arch}`;
}

export function detectVmCapability(): VmCapability {
  const platform = ociPlatform();
  const result = (reason: VmCapabilityReason, detail: string): VmCapability => ({
    available: reason === 'live',
    reason,
    detail,
    platform,
  });

  if (process.platform === 'linux') {
    if (!existsSync('/dev/kvm')) {
      return result('missing-kvm', '/dev/kvm is absent on this host (no KVM / nested virtualization)');
    }
    try {
      accessSync('/dev/kvm', constants.R_OK | constants.W_OK);
    } catch {
      return result('kvm-not-accessible', '/dev/kvm exists but is not read/writable by this user');
    }
  } else if (!(process.platform === 'darwin' && process.arch === 'arm64')) {
    return result('unsupported-platform', `real microVMs need Linux/KVM or Apple Silicon (this host: ${platform})`);
  }

  try {
    createRequire(import.meta.url).resolve('@machinen/runtime');
  } catch {
    return result('missing-runtime', '@machinen/runtime is not installed (optional peer of the plugin)');
  }

  return result('live', 'real microVM track enabled (KVM/HVF + @machinen/runtime present)');
}
```

```ts
// ---- whole-VM lane: ship the running process --------------------------------
// Phase 2 of the lifecycle story. The app-state arc above needed the guest to
// cooperate (dehydrate/rehydrate). This lane moves a microVM's WHOLE state ÔÇö
// heap, RNG, timers, kernel ÔÇö through plugin.publishMachine() and a
// `machinen+pull+...?artifact=vmstate` entry. The solver's progress is
// deliberately NOT covered by dehydrate(): only this lane can move it.
// (Inline the SolverProgress interface from Task 1 here ÔÇö server.ts inlines
// machine module shapes like CounterModule rather than importing across apps.)

const VM_NAME = 'vm_machine';
const VM_PUBLISH_PORT = Number(process.env.VM_PUBLISH_PORT ?? 3812);
// NOTE: SNAP_IMAGE no longer exists ÔÇö the lifecycle arc now pulls its image
// over HTTP (commit 887a849). Derive the bundle path directly:
const VM_IMAGE = path.resolve(import.meta.dirname, '../../remote/dist/index.js');
const VM_SNAP_DIR = path.resolve(import.meta.dirname, '../.machinen/web-vm-snapshots');
const VM_PUBLISH_DIR = path.resolve(import.meta.dirname, '../.machinen/web-vm-registry');
const VM_PULL_CACHE = path.resolve(import.meta.dirname, '../.machinen/web-vm-cache');

interface SolverModule {
  start(): Promise<SolverProgress>;
  stop(): Promise<SolverProgress>;
  progress(): Promise<SolverProgress>;
}

interface VmLaneState {
  phase: 'cold' | 'running' | 'published' | 'restored';
  busy: boolean;
  capability: VmCapability;
  /** Solver progress at the freeze point ÔÇö the witness the restore must beat. */
  frozenAt?: SolverProgress;
  published?: { digest: string; bytes: number; url: string; publishMs: number };
  pull?: { bytes: number; ms: number; cacheHit: boolean; digest?: string };
  timings: { bootMs?: number; publishMs?: number; restoreMs?: number; firstCallMs?: number };
}

const vmLane: VmLaneState = { phase: 'cold', busy: false, capability: vmCapability, timings: {} };
let vmPlugin: MachinenPlugin | undefined;
let vmPullPlugin: MachinenPlugin | undefined;
let vmHost: ReturnType<typeof createInstance> | undefined;
let vmPullHost: ReturnType<typeof createInstance> | undefined;

async function ensureVmInstances() {
  if (vmPlugin) return;
  // Lazy: non-KVM hosts never reach this line (routes 503 first), so
  // @machinen/runtime is only loaded where it can work.
  const { machinenDriver } = await import('@federated-compute/machinen-plugin');
  vmPlugin = machinenPlugin({
    driver: machinenDriver({ snapshotDir: VM_SNAP_DIR }),
    bootTimeoutMs: 180_000,
    // R2 ÔÇö verify the landed option name/shape against the runtime 2a API:
    publish: { dir: VM_PUBLISH_DIR, hostname: '127.0.0.1', port: VM_PUBLISH_PORT },
  });
  vmHost = createInstance({ name: 'vm_lane_host', remotes: [], plugins: [vmPlugin] });
  // bootTimeoutMs covers artifact fetch + restore until the R5 split lands;
  // 180s absorbs a multi-GB local pull with margin.
  vmPullPlugin = machinenPlugin({
    driver: machinenDriver({ snapshotDir: VM_SNAP_DIR }),
    bootTimeoutMs: 180_000,
    artifactCacheDir: VM_PULL_CACHE,
  });
  vmPullHost = createInstance({ name: 'vm_pull_host', remotes: [], plugins: [vmPullPlugin] });
  for (const p of [vmPlugin, vmPullPlugin]) {
    logHooks(p);
    recordWire(p);
  }
}

function vmLaneBody() {
  const { busy: _busy, ...rest } = vmLane;
  return rest;
}

/** Serialize lane steps; refuse honestly when the hardware can't run them. */
async function vmStep<T>(
  res: http.ServerResponse,
  allowed: VmLaneState['phase'][],
  step: () => Promise<T>,
): Promise<void> {
  if (!vmCapability.available) {
    return json(res, 503, {
      error: `live VM track unavailable: ${vmCapability.detail}`,
      capability: vmCapability,
    });
  }
  if (vmLane.busy) return json(res, 409, { error: 'a VM step is already running' });
  if (!allowed.includes(vmLane.phase)) {
    return json(res, 409, {
      error: `step not valid in phase "${vmLane.phase}" (expected ${allowed.join(' | ')})`,
    });
  }
  vmLane.busy = true;
  try {
    const extra = await step();
    json(res, 200, { ...vmLaneBody(), ...extra, wire: wire() });
  } finally {
    vmLane.busy = false;
  }
}

/** GET /api/vm/state ÔÇö phase + live solver progress when a VM is up. */
async function handleVmState(_req: http.IncomingMessage, res: http.ServerResponse) {
  let progress: SolverProgress | undefined;
  try {
    if (vmLane.phase === 'running') {
      progress = await (await vmHost!.loadRemote<SolverModule>(`${VM_NAME}/solver`))!.progress();
    } else if (vmLane.phase === 'restored') {
      progress = await (await vmPullHost!.loadRemote<SolverModule>(`${VM_NAME}/solver`))!.progress();
    }
  } catch {
    // VM mid-transition; state body still answers.
  }
  json(res, 200, { ...vmLaneBody(), progress });
}

/** POST /api/vm/boot ÔÇö boot the source microVM and start the solver. */
async function handleVmBoot(_req: http.IncomingMessage, res: http.ServerResponse) {
  await vmStep(res, ['cold', 'running', 'published', 'restored'], async () => {
    await ensureVmInstances();
    // Reset the arc: dispose any prior source/clone, wipe the pull cache so
    // each full run is honestly a cold pull.
    await vmPlugin!.disposeMachines();
    await vmPullPlugin!.disposeMachines();
    await rm(VM_PULL_CACHE, { recursive: true, force: true });
    vmLane.frozenAt = undefined;
    vmLane.published = undefined;
    vmLane.pull = undefined;
    vmLane.timings = {};
    vmHost!.registerRemotes([{ name: VM_NAME, entry: `machinen://${VM_IMAGE}` }], { force: true });
    const t0 = performance.now();
    const solver = (await vmHost!.loadRemote<SolverModule>(`${VM_NAME}/solver`))!;
    const progress = await solver.start(); // first call boots the VM
    vmLane.timings.bootMs = Math.round(performance.now() - t0);
    vmLane.phase = 'running';
    logEvent('ready', `${VM_NAME} booted as a real microVM in ${vmLane.timings.bootMs}ms ÔÇö solver running`);
    return { progress };
  });
}

/** POST /api/vm/publish ÔÇö freeze mid-flight, publish vmstate, kill the source. */
async function handleVmPublish(_req: http.IncomingMessage, res: http.ServerResponse) {
  await vmStep(res, ['running'], async () => {
    const solver = (await vmHost!.loadRemote<SolverModule>(`${VM_NAME}/solver`))!;
    vmLane.frozenAt = await solver.progress(); // the witness, captured just before the dump
    const t0 = performance.now();
    // R2 ÔÇö one verb: snapshot + content-addressed publish + endpoint start.
    const published = await vmPlugin!.publishMachine(VM_NAME);
    const publishMs = Math.round(performance.now() - t0);
    // The hero moment needs "the source is gone". The runtime resumes the
    // source after a dump; the DEMO kills it ÔÇö say so in the UI copy.
    await vmPlugin!.disposeMachines();
    vmLane.published = {
      digest: String((published as { digest: string }).digest),
      bytes: Number((published as { bytes: number }).bytes),
      url: String((published as { url: string }).url),
      publishMs,
    };
    vmLane.timings.publishMs = publishMs;
    vmLane.phase = 'published';
    logEvent('snapshot', `${VM_NAME} frozen mid-solve + published as vmstate (${vmLane.published.digest.slice(0, 19)}ÔÇŽ) ÔÇö source VM killed`);
    return {};
  });
}

/** POST /api/vm/restore ÔÇö pull the vmstate artifact, restore, prove continuity. */
async function handleVmRestore(_req: http.IncomingMessage, res: http.ServerResponse) {
  await vmStep(res, ['published'], async () => {
    // The deployment story is this one entry string (R1). Verify against the
    // landed publish URL shape: it must be the plugin endpoint base.
    const entry = `machinen+pull+http://127.0.0.1:${VM_PUBLISH_PORT}?artifact=vmstate&version=^1.0.0`;
    vmPullHost!.registerRemotes([{ name: VM_NAME, entry }], { force: true });
    const t0 = performance.now();
    const solver = (await vmPullHost!.loadRemote<SolverModule>(`${VM_NAME}/solver`))!;
    const progress = await solver.progress(); // first call pulls + restores
    vmLane.timings.restoreMs = Math.round(performance.now() - t0);
    vmLane.timings.firstCallMs = vmLane.timings.restoreMs;
    // R3 ÔÇö the artifact wire event for THIS request carries the pull stats.
    const pull = wire().find(
      (e): e is Extract<WireEvent, { type: 'artifact' }> =>
        e.type === 'artifact' && e.machine === VM_NAME,
    );
    vmLane.pull = {
      bytes: pull?.bytes ?? 0,
      ms: pull?.ms ?? 0,
      cacheHit: pull?.cacheHit ?? false,
      digest: pull?.digest,
    };
    const frozen = vmLane.frozenAt!;
    if (progress.iteration < frozen.iteration || progress.cacheSize < frozen.cacheSize) {
      throw new HttpError(500, `restored VM lost heap: iteration ${progress.iteration} < frozen ${frozen.iteration}`);
    }
    vmLane.phase = 'restored';
    logEvent('restore', `${VM_NAME} restored from pulled vmstate ÔÇö solver resumed at iteration ${progress.iteration} (frozen at ${frozen.iteration})`);
    return { progress, entry };
  });
}

/** POST /api/vm/reset ÔÇö tear the lane down and reclaim the multi-GB artifacts. */
async function handleVmReset(_req: http.IncomingMessage, res: http.ServerResponse) {
  if (vmLane.busy) return json(res, 409, { error: 'a VM step is already running' });
  vmLane.busy = true;
  try {
    await vmPlugin?.disposeMachines();
    await vmPullPlugin?.disposeMachines();
    await rm(VM_PULL_CACHE, { recursive: true, force: true });
    await rm(VM_PUBLISH_DIR, { recursive: true, force: true });
    vmLane.phase = 'cold';
    vmLane.frozenAt = undefined;
    vmLane.published = undefined;
    vmLane.pull = undefined;
    vmLane.timings = {};
    json(res, 200, vmLaneBody());
  } finally {
    vmLane.busy = false;
  }
}
// ----------------------------------------------------------------------------
```
