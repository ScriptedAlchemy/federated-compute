What you're touching

Mouse and keyboard go the whole way down: noVNC encodes them as RFB events, qemu turns them into usb-tablet absolute pointer reports, and Android 4.4 handles them as touch input. When the lab freezes the VM, your session drops — because the far end of this websocket literally stops existing — and reconnects against the thawed device, which kept drawing the same frame.

The pipe — browser to framebuffer

qemu speaks the websocket protocol natively, so there is no VNC gateway anywhere — the host server splices your browser's GET /vnc upgrade onto a forwarded TCP port as a dumb pipe, and the handshake completes inside the microVM.
